## Turning Everyday Appliances into a Seismic Shield

Traditional seismic monitoring networks are often composed of expensive, specialized equipment placed in specific geographic locations. **QuakeGuard** takes a different approach by leveraging the power of the Internet of Things (IoT) to turn common household appliances—like washing machines, refrigerators, and TVs—into a massive, distributed sensor array. By utilizing the ubiquity of connected devices, QuakeGuard creates a dense mesh of sensors capable of detecting and reporting earthquake activity with high granularity and speed.

## Intelligent Edge Sensing

At the heart of the system is the IoT Edge layer, built on the ESP32-C3 SuperMini. Each node is equipped with an ADXL345 triaxial accelerometer, sampling at 100 Hz. To distinguish between a passing truck and a genuine seismic event, the firmware implements a digital High-Pass Filter (HPF) to remove the constant influence of gravity and utilizes the **STA/LTA (Short Term Average / Long Term Average)** algorithm. This method, a staple in professional seismology, compares short-term signal energy to long-term background noise to trigger detections accurately.

The firmware is designed for reliability and performance, utilizing FreeRTOS tasks to manage concurrent operations. It uses statically allocated ring buffers for signal processing to ensure deterministic memory usage, which is critical for embedded stability during high-stress events.

## Security and Data Integrity

In an emergency alert system, data integrity is not optional; it is a requirement. QuakeGuard implements a robust security model to prevent spoofing and replay attacks. Every telemetry packet generated by an ESP32 node is cryptographically signed using **ECDSA NIST256p** (SHA256). 

When the backend receives a measurement, it performs a multi-step validation:
1. **API Key Verification**: Ensures the request comes from a known source.
2. **Signature Check**: Verifies the payload against the sensor's registered public key.
3. **Anti-Replay Window**: Validates the timestamp within a 60-second window to prevent old data from being re-injected into the system.

## Scalable Backend Architecture

Seismic events present a unique technical challenge: the "Thundering Herd" effect. When an earthquake occurs, thousands of sensors may attempt to report data simultaneously. QuakeGuard addresses this using an asynchronous, event-driven architecture. 

A FastAPI gateway receives the signed payloads and immediately pushes them into a **Redis Queue**. This decouples data ingestion from heavy processing. A background worker then consumes these messages, calculates a magnitude proxy based on Peak Ground Acceleration (PGA), and uses **PostgreSQL with PostGIS** to automatically assign the event to a geographic zone based on the sensor's coordinates. This spatial intelligence allows the system to target alerts to specific macro-regions, such as "Italy - North" or "Western Europe."

## Real-Time User Alerts

The final link in the chain is the React Native mobile application. Built with Expo and TypeScript, the app maintains a persistent WebSocket connection to the backend. When a critical event (Magnitude ≥ 4.5) is confirmed, the backend broadcasts an alert that triggers the mobile app within milliseconds. 

To ensure the user is notified even in high-stress situations, the app employs a custom **SOS haptic vibration pattern** and visual alerts. The interface includes a real-time monitor, a map of the sensor network, and an event history feed, providing users with a comprehensive view of seismic activity in their region.

## Stress Testing and Reliability

To ensure the system can handle real-world conditions, QuakeGuard includes a comprehensive stress test suite. This suite simulates hundreds of concurrent sensors firing data at the API, validating the rate limiters (50 req/s per IP), the Redis-based deduplication logic, and the end-to-end latency from the firehose ingestion to the database persistence. This rigorous testing ensures that when a real earthquake strikes, the architecture remains standing.
